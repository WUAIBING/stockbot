## Main Strategy Investigation Result

### 1. Root Cause Of Today's Buy/Add/Sell Anomaly

Confirmed:

- the DO machine runs on `UTC`
- the main-strategy execution layer used local `datetime.now()` for trade-window checks
- the day controller itself schedules by China market time, but the trader window gate compared against machine-local `UTC`

Observed impact on today's DO evidence:

- `09:36 / 10:28 / 13:28 add-position` all returned `window skipped`
- `14:50 buy-watch` retried repeatedly and each attempt returned `window skipped`
- this behavior is consistent with comparing `UTC` clock time against hard-coded `CST` trade windows

Conclusion:

- today's main-strategy execution trail was polluted by a timezone bug
- today's lack of buying cannot be treated as a clean proof that the strategy intentionally and correctly chose to stay fully defensive

### 2. What Was Fixed

Fixed in `v10_moni_trader.py`:

- introduced explicit market-time helpers based on `UTC+08`
- switched `buy / smart-sell / add-position` window checks to market time
- switched key "today/trade-date" checks in the main execution path to market date
- fixed `wait_for_today_decision_ready()` to use market date rather than machine-local date

Operational meaning:

- tomorrow's DO execution should no longer reject valid China-market windows simply because the server itself runs in `UTC`
